# 供应链履约Agent系统设计文档

> 日期：2026-05-23
> 场景优先级：1（供应链履约）→ 2（智能客服）→ 5（内部运营）→ 3（金融风控）→ 4（跨境）

---

## 一、设计目标

基于Claude Code纯原生能力，构建**供应链履约Agent系统**，采用Router + Sub-Agents架构，通过企业微信/钉钉机器人接入，Mock API模拟真实系统交互，快速验证Agent核心能力。

---

## 二、系统架构

```
用户（业务人员）
    ↓
企业微信/钉钉机器人（接入层）
    ↓
Router Agent（任务协调与路由）
    ↓
┌─────────────────────────────────────────────┐
│              Sub-Agent 专业能力层            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │库存查询  │ │路线规划  │ │异常处理  │    │
│  │Sub-Agent │ │Sub-Agent │ │Sub-Agent │    │
│  └──────────┘ └──────────┘ └──────────┘    │
│  ┌──────────┐ ┌──────────┐                  │
│  │通知推送  │ │数据记录  │                  │
│  │Sub-Agent │ │Sub-Agent │                  │
│  └──────────┘ └──────────┘                  │
└─────────────────────────────────────────────┘
    ↓
Mock API模拟层（ERP/WMS/TMS接口）
```

## 三、Agent职责设计

### 3.1 Router Agent

| 职责 | 说明 |
|------|------|
| 意图识别 | 解析用户/系统消息，判断任务类型（查询/调度/异常/通知） |
| 任务分发 | 根据任务类型路由到对应的Sub-Agent |
| 结果聚合 | 收集Sub-Agent返回结果，组装成用户可理解的响应 |
| 流程编排 | 处理多步骤任务（如：订单异常 → 库存查询 → 重路由 → 通知） |

### 3.2 Sub-Agent设计

| Sub-Agent | 核心能力 | 输入 | 输出 |
|-----------|---------|------|------|
| 库存查询Agent | 查询各仓库库存水位、可用量 | SKU/规格/数量 | 库存匹配方案 |
| 路线规划Agent | 计算最优物流路线、运费、时间 | 起始地/目的地/货物信息 | 推荐路线+成本 |
| 异常处理Agent | 检测异常、触发重路由、生成处理建议 | 异常事件（延迟/拒收等） | 处理方案+通知 |
| 通知推送Agent | 统一消息推送（微信/钉钉/邮件） | 消息内容+接收人+渠道 | 发送状态 |
| 数据记录Agent | 记录操作日志、更新状态 | 操作类型+数据 | 状态更新确认 |

## 四、技术实现路径

| 层级 | 实现方式 |
|------|---------|
| **Skill定义** | 定义每个Agent的prompt模板、工具集、SOP |
| **Agent实现** | 使用Claude Code内置Agent能力 |
| **MCP连接** | 通过MCP Server连接Mock API |
| **知识库** | RAG存储SOP、历史案例、业务规则 |
| **可观测性** | 日志Trace记录每步操作、耗时、工具调用 |

## 五、交付范围（Phase 1 - Mock模式）

### 5.1 第一批交付物

1. **Router Agent** - 意图识别 + 任务路由
2. **库存查询Sub-Agent** - Mock API调用
3. **路线规划Sub-Agent** - Mock API调用
4. **异常处理Sub-Agent** - 事件触发 + 处理流程
5. **企业微信机器人接入** - 消息接收与响应

### 5.2 验收标准

| 指标 | 目标值 |
|------|--------|
| 任务完成率 | ≥ 85% |
| 意图识别准确率 | ≥ 90% |
| 平均响应时间 | < 10s |
| 工具调用成功率 | ≥ 95% |

## 六、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent框架 | Claude Code原生Agents | 代码可控，调试便捷 |
| 工作流定义 | Skills + Prompt模板 | 统一管理，易维护 |
| 系统连接 | MCP Server | 标准协议，易扩展 |
| 消息接入 | 企业微信/钉钉 | B端业务人员熟悉 |
| Mock API | FastAPI + JSON | 轻量，快速验证 |

## 七、风险与治理

| 风险 | 应对策略 |
|------|---------|
| 幻觉/错误决策 | 关键节点HITL人工确认 + 规则引擎兜底 |
| 上下文丢失 | 会话状态持久化 + 业务ID透传 |
| 系统对接阻力 | API网关统一封装 + 先读后写 |
| 成本与延迟 | 模型路由 + Prompt缓存 |
